## Срединный путь: Концепция идеального вывода

Чтобы продукт действительно решал боль пользователя (сберечь время и силы), вывод должен строиться по принципу **«Светофор + План действий»**.

2. **Блок «Сравнение требований» (Экран 2):** Наглядная таблица (из Варианта 2).


3. **Блок «Как адаптировать резюме» (Экран 3):** Конкретные советы по рефреймингу (из Варианта 1), если пользователь всё же решил подаваться.



Всю «аналитику» (архетипы роли, боли компании, вытаскивание требований) нужно **спрятать под капот** — ИИ должен использовать её для формирования выводов, но не вываливать на пользователя.
