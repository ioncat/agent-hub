## Strategic Context

Third pipeline step. Takes a fetched + analysed vacancy (JD.md + JD_analysis.md) and produces
the tailored CV files: `[Name]_CV.md` and `[Name]_CV.pdf`. Two-phase LLM flow:
Phase 3 generates a raw draft (hidden from user); Phase 3.5 self-reviews and produces the final
revised CV + a critique block shown to the user.

Introduces `CVAdapter` вАФ subprocess wrapper for `cv_to_pdf.py` in the callback-cv repo.

---

## Goal

`tools/cv_generate.py` exposes `cv_generate(ctx, vacancy_id, language) вЖТ str`.
Registered in `agent.py` ToolRegistry.

## Phases

| Phase | Prompt file | Input | Output |
|-------|-------------|-------|--------|
| Phase 3 | `phase3_cv_draft.md` | JD + analysis + language + name | Raw CV draft (saved to `CV_draft_p3.md`, hidden) |
| Phase 3.5 | `phase3_5_review.md` | JD + analysis + Phase 3 draft | Self-review block + final revised CV |

Phase 3.5 output is split into two parts via `_split_review_and_cv`:
- **Review block**: `CV SELF-REVIEW...вЬЕ Keep as is` section вЖТ shown to user + appended to `JD_analysis.md`
- **Final CV**: name/headline/contacts block + SUMMARY + EXPERIENCE + ... вЖТ saved as `[Name]_CV.md`

Split anchor: `SUMMARY` on its own line (mandatory in Phase 3 template).
Walk back to last `\n\n` (with `rstrip("\n")` guard) to include name/headline in CV part.

## CVAdapter

`adapters/cv_adapter.py` вАФ subprocess wrapper for `cv_to_pdf.py`.

- Resolves Python: `callback-cv/venv/Scripts/python.exe` (Win) or `venv/bin/python` (Unix)
- Falls back to `sys.executable` with a warning if venv not found
- 30s subprocess timeout
- Raises `CVAdapterError` on non-zero exit or missing PDF
- PDF generation is **best-effort** in `cv_generate` вАФ failure logged, not fatal

## User Stories

### US-901: Generate CV for analysed vacancy

**Given** vacancy has status `analyzed`, JD.md + JD_analysis.md exist
**When** `cv_generate(ctx, 42)` runs
**Then**:
- Phase 3 draft generated (hidden, saved to `CV_draft_p3.md`)
- Phase 3.5 self-review run; output split into review block + final CV
- `[Name]_CV.md` saved
- `[Name]_CV.pdf` generated
- Review block appended to `JD_analysis.md`
- Vacancy status вЖТ `cv_generated`
- Telegram reply: "вЬЕ CV –≥–Њ—В–Њ–≤" + full self-review critique

### US-902: JD_analysis.md missing

**Given** analysis not yet run (status `fetched`)
**When** `cv_generate` runs
**Then** return user-friendly error "вЪ†пЄП JD_analysis.md –љ–µ –љ–∞–є–і–µ–љ"

### US-903: PDF generation fails

**Given** callback-cv venv missing or fpdf not installed
**When** `cv_generate` runs
**Then** CV.md is still saved, warning returned in message, no exception raised

### US-904: LLM error on Phase 3

**Given** Claude returns `LLMError` on Phase 3 call
**When** `cv_generate` runs
**Then**:
- `pipeline_runs` row for phase3 marked `error`
- No CV files written
- Status not updated
- User-friendly error returned

### US-905: LLM error on Phase 3.5

**Given** Phase 3 succeeds, Phase 3.5 raises `LLMError`
**When** `cv_generate` runs
**Then**:
- `pipeline_runs` row for phase3_5 marked `error`
- `[Name]_CV.md` NOT written
- Status not updated
- User-friendly error returned
